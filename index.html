Security

Privacy isn't a feature.
It's the foundation.

Every security decision in Monolith was made assuming the worst case — someone gets physical access to your hardware. Your data is still safe.

⚠

Master key lives on the same drive

Monolith encrypts your data at rest using a master key — but that key is stored in the same directory as the application. If someone steals the physical drive, they could potentially access both the key and the encrypted data together.

Recommendation: encrypt the drive Monolith runs on. Full-disk encryption (like LUKS on Linux) means the master key and your data are protected by a second layer — the drive is unreadable without the disk password, regardless of who has physical access to it.

Black Box solves this by default. Every Black Box ships with full-disk encryption on the SD card and M.2 SSD baked in at the hardware level. If the drive is removed or the entire unit is stolen, the data on it is cryptographically unreadable. No password, no data.

Build pipeline
NoMad Software in the repo today

Monolith in your pocket.
No install. No trace.

NoMad is Monolith on a USB stick. Plug into a Windows or Linux PC, run one script, and Monolith is live in your browser — reading and writing only to the stick. When you're done, pull it. Nothing stays behind.

How it works
1

Plug in the USB

Monolith lives on the stick. Nothing gets installed on the host machine.

2

Run the script

On Windows run launch.bat. On Linux run bash launch.sh. No setup, no configuration.

3

Your browser opens

The launcher opens a single-use launch link on localhost:5000. Monolith is running, fully functional, visible only to that PC.

4

Pull and go

Close the tab and Nomad shuts itself down moments later. Pull the stick — your data leaves with you.

What makes it different
◈
Single-use launch — every boot mints a one-time launch token and a fresh session key. A cookie left on a shared PC is useless afterwards.
⬡
Auto-shutdown — close the tab and Nomad shuts down on its own about 25 seconds later, telling the browser to clear site data on the way out.
◻
Data stays on USB — your financial data is read from and written only to the stick, never the host filesystem.
⊕
Cross-platform — one stick works on Windows (launch.bat) and Linux (launch.sh). No separate versions.
⟳
Try before you commit — the perfect way to test Monolith before a permanent install. Export your data as JSON and import it later.
ℹ

One requirement today: the host PC needs Python 3.10+. The pre-built NoMad stick bundles its own portable runtime so the host needs nothing at all — that's what "coming soon" means.

⚠ Honest disclaimer

NoMad is designed to minimize its footprint on the host machine and makes every effort to leave no trace. However, we cannot guarantee zero traces in all cases. Operating systems may write swap files, temp caches, or memory pages to disk under conditions outside our control. If absolute zero-trace operation is a hard requirement, use NoMad on an air-gapped machine or one with an encrypted drive.
